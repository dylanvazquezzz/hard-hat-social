**Contractor fetch:** On mount (useEffect with `[]` dependency), fetch all approved contractors using the browser supabase client:
```typescript
supabase
  .from('contractors')
  .select('id, full_name, trade, location_city, location_state')
  .eq('status', 'approved')
  .order('full_name')
  .then(({ data }) => setContractors((data as Contractor[]) ?? []))
```
Import `supabase` from `@/lib/supabase`. Do NOT use `getSupabaseAdmin()` here — this is a client component and that import will fail the build.

**State:** `contractors: Contractor[]`, `search: string`, `submitting: boolean`, `selectedId: string | null`.

**Client-side filter:**
```typescript
const filtered = contractors.filter(
  (c) =>
    search === '' ||
    c.full_name.toLowerCase().includes(search.toLowerCase()) ||
    c.trade.toLowerCase().includes(search.toLowerCase())
)
```

**On confirm:** Call `markHired(jobId, selectedId!)` (imported from `@/app/jobs/actions`). Set `submitting: true` before the call. After the call resolves, call `onClose()`. The page re-renders via `revalidatePath` from the server action — no client-side navigation needed.

**Modal layout:** Fixed overlay backdrop (`fixed inset-0 z-50 flex items-center justify-center bg-black/60`). Inner panel: `max-w-md w-full mx-4 bg-slate-900 border border-slate-700 rounded-lg p-5 shadow-xl`. Structure inside the panel:
- Heading: "Select a Contractor to Hire"
- Search input (text, placeholder "Search by name or trade") — controls `search` state
- Scrollable contractor list (`max-h-72 overflow-y-auto space-y-2`) — each item is a button that sets `selectedId`; highlight the selected item with `border-amber-500` ring or background change
- Each list item shows: full_name (bold), trade + city/state (text-sm text-slate-400)
- Footer row: "Cancel" button (calls `onClose()`) and "Confirm" button (amber-500, disabled if `!selectedId || submitting`, shows "Hiring..." when submitting)

Dismiss on backdrop click: clicking the backdrop overlay div calls `onClose()`.
