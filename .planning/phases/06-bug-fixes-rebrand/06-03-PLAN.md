<task type="checkpoint:human-action" gate="blocking">
  <action>Start Resend DNS and GoDaddy DNS configuration. This has a 24-48 hour propagation window — it must happen before the Vercel domain can be verified and before any email from hardhatsocial.net will reach inboxes.</action>
  <instructions>
    **Step 1 — Add hardhatsocial.net to Resend (do this first to get DNS record values):**
    1. Go to https://resend.com/domains
    2. Click "Add Domain"
    3. Enter `hardhatsocial.net`
    4. Resend will display the required DNS records — copy them exactly (do not guess these values)
    5. You will need: an SPF TXT record and a DKIM TXT record (on `resend._domainkey.hardhatsocial.net`)

    **Step 2 — Add hardhatsocial.net to Vercel to get DNS values:**
    1. Go to your Vercel project > Settings > Domains
    2. Click "Add Domain" and enter `hardhatsocial.net`
    3. Also add `www.hardhatsocial.net`
    4. Vercel will show the required DNS records — the exact IP for the A record and CNAME target
    5. Note: Vercel's current anycast IP is `76.76.21.21` but ALWAYS use the value Vercel shows you — do not assume

    **Step 3 — Add all DNS records in GoDaddy:**
    Go to https://dcc.godaddy.com/manage (or https://dns.godaddy.com) for hardhatsocial.net DNS management.

    Add these records (exact values from Vercel and Resend dashboards):
    - `A` record: Host = `@`, Points to = [Vercel's IP, e.g. 76.76.21.21], TTL = 600
    - `CNAME` record: Host = `www`, Points to = `cname.vercel-dns.com.` (include trailing dot if GoDaddy requires it), TTL = 600
    - Resend SPF: `TXT` record on the host that Resend specifies (likely `@` or `send`), Value = [Resend SPF value], TTL = 600
    - Resend DKIM: `TXT` record: Host = `resend._domainkey`, Value = [Resend DKIM value], TTL = 600

    Note: GoDaddy's DNS UI strips the domain suffix from Host fields automatically. If Resend says the record is for `resend._domainkey.hardhatsocial.net`, enter just `resend._domainkey` in the Host field.

    **DNS propagation takes 24-48 hours.** After adding these records, you can continue with the non-DNS steps below while waiting.
  </instructions>
  <resume-signal>Type "dns-started" once all GoDaddy DNS records are saved. You do not need to wait for propagation before continuing to the next steps.</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Complete Vercel domain setup, update NEXT_PUBLIC_APP_URL, and verify deployment serves hardhatsocial.net.</action>
  <instructions>
    **Prerequisite:** DNS propagation must have started (Step 1 above). You don't need full propagation — Vercel can verify the domain as soon as DNS is propagating. This may take a few minutes to a few hours.

    **Step 1 — Verify Vercel domain configuration:**
    1. Vercel project > Settings > Domains
    2. Both `hardhatsocial.net` and `www.hardhatsocial.net` should have been added already
    3. Wait until Vercel shows a green checkmark or "Valid Configuration" — this means DNS has propagated enough
    4. If it shows "Invalid Configuration" after 2+ hours, double-check GoDaddy DNS records match exactly what Vercel specified
    5. Vercel auto-provisions a TLS certificate — this also requires DNS propagation

    **Step 2 — Update NEXT_PUBLIC_APP_URL in Vercel:**
    1. Vercel project > Settings > Environment Variables
    2. Find `NEXT_PUBLIC_APP_URL` (or add it if it doesn't exist)
    3. Set value to `https://hardhatsocial.net`
    4. Set for: Production, Preview, Development environments
    5. Also check `RESEND_FROM_EMAIL` — if it exists and still says `noreply@contractorsconnect.com`, update to `noreply@hardhatsocial.net`
    6. Also add `NEXT_PUBLIC_APP_URL=https://hardhatsocial.net` to your local `.env.local` file if you have one

    **Step 3 — Redeploy:**
    1. Vercel project > Deployments
    2. Find the latest production deployment
    3. Click the three-dot menu > "Redeploy" (this picks up the new env vars)
    4. Or push a commit to main to trigger a new deployment

    **Step 4 — Verify:**
    1. Visit https://hardhatsocial.net in your browser
    2. The app should load with "Hard Hat Social" branding
    3. Open DevTools > Application > Local Storage — check that there are no errors
    4. Run in terminal: `curl -I https://hardhatsocial.net` — should return HTTP/2 200
  </instructions>
  <resume-signal>Type "domain-live" once https://hardhatsocial.net serves the app correctly with a green TLS padlock. Include what `curl -I https://hardhatsocial.net` returned.</resume-signal>
</task>
